== Purpose

This repository contains sanctioned individuals, entities, and vessels from the
Australian Department of Foreign Affairs and Trade (DFAT) Consolidated List.

== Source

* **Authority**: Australia Department of Foreign Affairs and Trade (DFAT)
* **URL**: https://www.dfat.gov.au/international-relations/security/sanctions/consolidated-list
* **XLSX Download**: https://www.dfat.gov.au/sites/default/files/Australian_Sanctions_Consolidated_List.xlsx
* **Format**: XLSX
* **Update Frequency**: As updated by DFAT
* **Country Code**: AU

== Legal Basis

Australia implements sanctions under multiple legal frameworks:

* *Autonomous Sanctions Act 2011*
* *Autonomous Sanctions Regulations 2011*
* *Charter of the United Nations Act 1945*
* Various country-specific regulations

== Data Contents

The DFAT list contains:

* **Individuals**: Natural persons subject to sanctions
* **Organizations**: Companies and entities subject to sanctions
* **Vessels**: Ships subject to maritime sanctions

== Source Data Format

The processed YAML files in `processed/entities/` follow this structure:

=== Organization Entity
[source,yaml]
----
id: '8577'
type: organization
names:
  - full_name: "EXAMPLE COMPANY LTD"
    is_primary: true
    script: Latn
  - full_name: "示例公司"
    is_primary: false
    script: Hani
source_references:
  - source_code: au
    reference_number: '8577'
organization_details:
  country_of_registration: Russia
remarks: 'Additional information about the entity'
----

=== Person Entity
[source,yaml]
----
id: '1234'
type: person
names:
  - full_name: "IVAN IVANOV"
    is_primary: true
    script: Latn
source_references:
  - source_code: au
    reference_number: '1234'
person_details:
  dates_of_birth:
    - year: 1970
      month: 1
      day: 15
  nationalities:
    - Russia
remarks: 'Additional information about the person'
----

=== Vessel Entity
[source,yaml]
----
id: '5678'
type: vessel
names:
  - full_name: "EXAMPLE VESSEL"
    is_primary: true
    script: Latn
source_references:
  - source_code: au
    reference_number: '5678'
vessel_details:
  imo_number: '1234567'
  flag: Russia
remarks: 'Additional information about the vessel'
----

=== Field Definitions

[cols="1,4"]
|===
| Field | Description

| `id`
| Unique identifier for the entity

| `type`
| Entity type: `person`, `organization`, or `vessel`

| `names`
| Array of name variants with `full_name`, `is_primary`, and `script`

| `source_references`
| Array linking to original source with `source_code` and `reference_number`

| `person_details`
| Person-specific fields (dates_of_birth, nationalities, etc.)

| `organization_details`
| Organization-specific fields (country_of_registration, etc.)

| `vessel_details`
| Vessel-specific fields (imo_number, flag, etc.)

| `remarks`
| Additional textual information

|===

== Harmonized Data Format

After harmonization, entities are transformed into JSON-LD format for the knowledge graph.

=== Entity (JSON-LD)
[source,json]
----
{
  "@context": "https://www.ammitto.org/schema/v1/context.jsonld",
  "@id": "https://www.ammitto.org/entity/au/8577",
  "@type": "OrganizationEntity",
  "names": [
    {
      "@type": "NameVariant",
      "fullName": "EXAMPLE COMPANY LTD",
      "script": "Latn",
      "isPrimary": true
    }
  ],
  "organizationDetails": {
    "@type": "OrganizationDetails",
    "countryOfRegistration": "RU"
  },
  "remarks": "Additional information",
  "sanctionEntries": [
    {
      "@id": "https://www.ammitto.org/entry/au/8577",
      "@type": "SanctionEntry",
      "authority": {
        "@id": "https://www.ammitto.org/authority/au",
        "name": "Australia DFAT"
      },
      "regime": {
        "@type": "SanctionRegime",
        "code": "AU_RUSSIA",
        "name": "Russia"
      },
      "effects": [
        {
          "@type": "SanctionEffect",
          "effectType": "asset_freeze",
          "scope": "full"
        },
        {
          "@type": "SanctionEffect",
          "effectType": "entry_ban",
          "scope": "full"
        }
      ],
      "status": "active",
      "period": {
        "@type": "TemporalPeriod",
        "listedDate": "2022-03-01"
      }
    }
  ]
}
----

=== Sanction Entry Fields

[cols="1,4"]
|===
| Field | Description

| `effects`
| Array of sanction effects (asset_freeze, entry_ban, trade_restriction, etc.)

| `regime`
| Sanctions regime (Russia, DPRK, Iran, etc.)

| `status`
| Entry status (active, delisted, expired)

| `period`
| Temporal information (listedDate, effectiveDate, expiryDate)

| `legalInstruments`
| Legal basis for the sanction

|===

== Knowledge Graph Integration

This repository's processed YAML data feeds into the Ammitto knowledge graph.

When the harmonization process runs:

[source,shell]
----
# Auto-detect all data-* repos and harmonize
ammitto harmonize --scan --sources-dir /path/to/repos --output-dir ./api/v1
----

The au data is transformed into:

* Individual JSON-LD nodes in `api/v1/node/entity/au/{id}.jsonld`
* Sanction entries in `api/v1/node/entry/au/{id}.jsonld`
* Lightweight search index entries
* Facet counts for filtering
